# 奇妙集市功能设计说明书

## 1. 项目定位

奇妙集市是一个同时面向移动端微信网页与 PC 浏览器的电商网站，分为导航页、用户端商城和管理员后台三部分。系统以 PHP 8 + MySQL + Tailwind CSS + Alpine.js 为正式实现栈，通过独立的会员同步服务与会员系统数据库联动，实现会员等级、余额、折扣和消费日志的同步。

## 2. 需求优化与冲突修订

### 2.1 技术栈修订
- 原始草案要求后端语言从 Go / Java / PHP 中选择，同时又要求使用 SQLAlchemy ORM。两者存在技术冲突。
- 正式版采用 PHP 8.2+ 作为后端语言，数据库访问采用 PDO 预处理与白名单动态条件，达到“禁止原生 SQL 拼接”的安全目标。
- 前端采用 Tailwind CSS + Alpine.js，满足移动端友好、交互动画、后台复杂表单和弹窗管理需求。

### 2.2 订单取消规则修订
- 原始草案要求“已发货前用户可主动取消”，但同时不开发退款流程。
- 正式版调整为：
  - 待付款订单支持用户直接取消。
  - 已支付未发货订单允许用户发起取消。
  - 余额支付订单取消时，系统即时补偿会员余额并写入商城与会员联动记录。
  - 微信支付订单取消时，系统将订单关闭并标记为待人工退款，不在本期内实现完整退款工作流。

### 2.3 省市区数据策略
- 系统内置中国大陆省级行政区数据及代表性城市/区县三级联动结构，已满足页面三级联动和接口联调。
- 若进入正式生产并需要完整国标区划，可直接替换 `public/assets/data/china_regions.json`。

## 3. 信息架构与路由规划

### 3.1 路由约束
- `/`：导航页，不属于商城首页。
- `/mall`：商城首页。
- `/mall/*`：所有商城与后台路由，满足“除导航页外全部使用 mall 前缀”的要求。

### 3.2 页面结构
- 导航页：奇妙集市、知识星球两个图标菜单，展示热门活动、商品上新、推荐课程。
- 用户端：商城首页、商品详情、购物车、结算页、订单结果、用户中心、微信绑定页。
- 管理端：数据看板、商品管理、分类管理、订单管理、用户管理、活动管理、系统设置、日志中心。

## 4. 用户端功能

### 4.1 游客权限
- 游客可访问导航页、商城首页、商品详情页。
- 游客不可加入购物车、下单、支付、管理地址、查看订单、绑定微信或进入后台。

### 4.2 用户中心
- 个人信息：昵称修改、密码修改。
- 收货地址：增删改查、默认地址、省市区三级联动。
- 历史订单：全部、待付款、已付款、已发货、已关闭分组查看。
- 我的钱包：实时读取会员等级、会员余额，展示商城消费记录。

### 4.3 商品系统
- 商品列表：分页、排序、品牌筛选、价格区间筛选、Quick View。
- 商品详情：多图展示、SKU 联动选择、库存校验。
- 课程商品：课程与实物商品统一建模，以 `is_course` 区分。

### 4.4 购物车与订单
- 购物车支持数量步进、失效商品标记、实时价格计算。
- 下单流程：购物车 → 确认订单 → 支付 → 结果页。
- 库存策略：下单即扣减库存；15 分钟内未支付自动关闭订单并回滚库存。
- 金额计算：商品价 → 会员折扣 → 最终价。

### 4.5 支付
- 余额支付：调用会员系统兼容逻辑扣减余额，并写入 `menmberdetail` 与 `menmberdetail_log`。
- 微信支付：提供 JSAPI/H5 下单封装、支付配置测试按钮、回调入口。
- 幂等策略：订单支付状态为 `pending_payment` 才可进入支付成功流转。

## 5. 会员系统联动设计

### 5.1 联动边界
- 商城库只保存 `membership_member_id`，不冗余保存会员余额与会员等级。
- 商城读取会员信息时实时查询会员库 `member`、`classes`。
- 商品是否参与会员折扣由商城商品字段 `support_member_discount` 控制。

### 5.2 余额支付复用逻辑
- 复用会员系统 Python 版核心逻辑：
  - 扣费前查询会员余额。
  - 扣费成功后同时写入 `menmberdetail` 和 `menmberdetail_log`。
  - 商品名称为订单内商品名称逗号拼接。
  - 备注固定为“电商网站自助下单”。
- 若商城侧在会员扣费成功后提交失败，则触发商城补偿充值，保证数据尽量回滚一致。

### 5.3 管理端会员协同
- 后台用户可绑定会员 ID。
- 后台可新建会员、修改会员信息、手动调增/调减余额。
- 会员等级折扣由 `classes.foff` 直接驱动商城折扣计算。

## 6. 管理后台功能

### 6.1 数据看板
- 今日订单数。
- 今日 GMV。
- 今日新增用户数。
- 库存预警数。
- 近 7 日销售额图表。

### 6.2 商品与分类
- 商品支持新增、编辑、批量上下架、价格批量调整。
- 编辑页分为“商品信息 / 商品详情”页签。
- 商品详情页支持 TinyMCE 富文本与拖拽上传。
- 分类支持三级树结构与拖拽排序。

### 6.3 订单与用户
- 订单支持分组查看、发货、关闭。
- 用户支持后台创建，不开放前台注册。
- 后台记录手机号、OpenID、会员绑定 ID。

### 6.4 系统设置
- 会员系统配置：地址、数据库访问信息、测试按钮。
- 微信支付配置：AppID、商户号、APIv3 密钥、证书内容、测试按钮。
- 微信服务号配置：AppID、Secret、测试按钮。
- 日志配置：最小日志级别、保留时长、单文件大小。

## 7. 安全设计
- 使用 PDO 预处理，禁止 SQL 拼接注入。
- 富文本在服务端通过 DOM 白名单清洗，过滤脚本注入。
- 所有写操作启用 CSRF Token。
- 登录接口按 IP 限流：5 次 / 分钟。
- 密钥、密码等敏感设置以 AES-256 方式加密存储。
- Redis 用于限流与订单超时辅助键。
- 系统日志按 `debug / info / warning / error / critical` 分级。

## 8. 微信能力接入点
- OAuth 授权绑定：`/mall/bind/wechat` 与 `/mall/api/wechat/callback`。
- JSAPI / H5 支付封装：`WechatService::createPayOrder()`。
- 支付回调：`/mall/api/wechat/notify`。
- 服务号模板通知：管理员付款/取消通知，用户下单/发货/完成/关闭通知。

## 9. 静态资源与图片策略
- 项目不直接生成图片。
- 所有设计位统一预留 `.webp` 路径。
- 同路径下提供 `.webp.txt` AI 提示词文件，方便后续批量补图。
